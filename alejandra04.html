Protección contra Ataques Informáticos (Unidad 1)

Método de Protección Descripción Efectividad Ejemplo Imagen
Concientización y Educación Capacitar a usuarios para reconocer correos, mensajes o sitios sospechosos que intenten suplantar identidades. Alta, reduce significativamente los casos de phishing al aumentar la desconfianza en mensajes no verificados. Entrenamiento empresarial para identificar correos falsos que piden credenciales.
Antivirus y Antimalware Software que detecta, bloquea y elimina programas maliciosos como virus, troyanos o gusanos. Muy alta, aunque depende de actualizaciones regulares y la calidad del software. Instalación de herramientas como Kaspersky o Malwarebytes para escanear y eliminar amenazas.
Copias de Seguridad Realizar respaldos regulares de datos en ubicaciones seguras para recuperar información tras un ataque. Alta, permite restaurar datos cifrados por ransomware sin pagar rescate. Respaldos automáticos en la nube o discos externos desconectados tras cada respaldo.
Firewalls y Balanceadores Configurar firewalls y sistemas de balanceo de carga para filtrar tráfico malicioso y mitigar saturaciones. Alta contra DDoS, reduce el impacto de tráfico masivo en servidores. Uso de servicios como Cloudflare para dispersar tráfico en ataques DDoS.
Detección de Spyware Utilizar herramientas antimalware específicas para detectar y eliminar programas que monitorizan actividades. Moderada a alta, efectiva si se combina con navegación segura y permisos restringidos. Escanear dispositivos con software como Spybot para eliminar spyware oculto.
Validación de Entradas Implementar sanitización y validación estricta en formularios y consultas para evitar inyecciones maliciosas. Muy alta, previene casi completamente las inyecciones SQL si se implementa correctamente. Uso de consultas parametrizadas en aplicaciones web para evitar comandos SQL maliciosos.
Cifrado y VPN Usar protocolos seguros (HTTPS, TLS) y redes privadas virtuales para proteger datos en tránsito. Alta, dificulta la intercepción de datos en ataques MitM, especialmente en redes públicas. Conexión a una VPN como NordVPN en una red Wi-Fi pública para cifrar el tráfico.
Verificación de Proveedores Auditar y verificar la seguridad de software y servicios de terceros antes de integrarlos. Moderada a alta, reduce riesgos si se implementan auditorías rigurosas y actualizaciones controladas. Revisión de bibliotecas de código abierto antes de incluirlas en proyectos.
