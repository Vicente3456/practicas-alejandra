Ataques Informáticos (Unidad 1)

Ataque Descripción Consecuencias Ejemplo / Caso real Imagen
Phishing Mensajes (email, SMS, redes) o páginas que suplantan identidades para convencer al usuario de revelar credenciales o datos sensibles. Robo de identidad, acceso no autorizado a cuentas bancarias o correo, fraude financiero y robo de información. Correo que simula ser del banco pidiendo verificar la cuenta mediante un link falso que captura la contraseña.
Malware Software malicioso (virus, gusanos, troyanos) diseñado para dañar, espiar o controlar equipos y redes. Corrupción de archivos, pérdida de rendimiento, robo de datos confidenciales, puertas traseras persistentes. Descarga de software pirata que incluye malware; ejecución de adjuntos maliciosos.
Ransomware Tipo de malware que cifra archivos o bloquea sistemas y exige rescate (a menudo en criptomonedas) para devolver acceso. Pérdida de datos, parálisis operativa, costos por recuperación y daños reputacionales. Ataques a hospitales y municipalidades donde se cifraron sistemas críticos (p.ej. casos conocidos en 2010s–2020s).
DDoS (Ataque de denegación de servicio) Envío masivo y coordinado de solicitudes desde muchos equipos para saturar recursos y dejar servicios inaccesibles. Interrupción de servicios web, pérdidas económicas, reputación dañada y costes en mitigación. Ataque contra infraestructuras de DNS (p.ej. Dyn, 2016), que afectó a grandes plataformas online.
Spyware Software escondido que monitoriza la actividad del usuario y envía información sin que el usuario lo sepa. Robo de contraseñas, seguimiento de navegación, espionaje corporativo o personal. Apps móviles o extensiones que envían historiales y credenciales a terceros.
Inyección SQL Inserción de comandos maliciosos en campos de entrada para manipular las consultas a la base de datos. Exposición o modificación de datos, acceso no autorizado a información sensible, pérdida de integridad. Formularios web que no validan entradas permitiendo extraer tablas de la base de datos.
Man‑in‑the‑Middle (MitM) El atacante intercepta y/o modifica la comunicación entre dos partes sin que estas lo detecten. Robo de credenciales, manipulación de mensajes, robo de datos en tránsito y compromisos de sesiones. Intercepción de tráfico en una Wi‑Fi pública no protegida para capturar contraseñas o cookies de sesión.
Supply‑Chain / Ataque a la cadena de suministro Comprometer software, servicios o componentes de terceros que luego se distribuyen a múltiples víctimas. Compromiso masivo de clientes/usuarios a través de una actualización o biblioteca infectada; impacto a gran escala. Inserción de código malicioso en un paquete de actualización legítimo (ej.: incidentes donde bibliotecas o proveedores fueron comprometidos).
